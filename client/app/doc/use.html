Generating Pages / Routes Generate ReST API Endpoints yo angular-fullstack:endpoint task Define ReST Models (server/api/task/task.model.js)

The model defined here will automatically be migrated into database. var TaskSchema = new mongoose.Schema({ name: String, info: String, active: Boolean, image: String, category: String, updatedAt: String, createdAt: String, modifiedBy: String });

Generate the front end route

yo angular-fullstack:route task

Add CRUD table directive into HTML (task.html) crud-table api='task' fields='fields' Define the fields for CRUD table directive (task.controller.js)

$scope.fields = [ {field: 'image',dataType: 'image'}, {field: 'name'}, {field: 'info', noSort: true, noEdit: true}, {field: 'category', dataType: 'select', options: ['Shopping', 'Promotions', 'SEO', 'Developments']}, {field: 'active', dataType: 'boolean'} ];

Optionally you may define the page title (task.js)

angular.module('crudApp') .config(function ($stateProvider) { $stateProvider .state('task', { url: '/task', templateUrl: 'app/task/task.html', controller: 'TaskCtrl', title: 'Easy Task Manager Single Page APP' }); });

Make the route accesible only to logged in users (task.js)

Page title can be added into the route specification angular.module('crudApp') .config(function ($stateProvider) { $stateProvider .state('task', { url: '/task', templateUrl: 'app/task/task.html', controller: 'TaskCtrl', title: 'Easy Task Manager Single Page APP', authenticate: true }); });

Additional Options

The following options are available for CRUD table directive which enables us to control the way CRUD table be generated
Add these as attributes into html tag
Example: crud-table api='user' options='admin.options' noDelete

  • heading, title: defines column title of the generated table
  • noSort: Sort option for the column will be disabled
  • noAdd: User will not be able to add a new record
  • noEdit: Edit will be disabled in the generated table
  • noExport: Export option will not be generated
  • Data Types: The following data types are available for CRUD Table directive. The add, edit, sort decisions depends on these options.
    • Default data type is text
    • number, numeric, float, integer: The column data type will be presented as integer. While adding or editing, the input field won't allow text or special characters
    • calendar, date: This option will restrict the column content to date. The editing page will contain a calendar in place of simple input field
    • multiline, textarea: This will create a multiline input field for table data input
    • dropdown, select, option : This requires an additional option named 'options' which is an array containing options that will be available to the end user
    • image, photo, picture : Accepts an image url as inoput and displays the text into the table content
